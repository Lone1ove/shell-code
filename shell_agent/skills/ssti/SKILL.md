# 服务端模板注入（SSTI）

适用于 Jinja2、Twig、Freemarker、Velocity 等服务端模板引擎。

## 触发时机

- 响应中出现 `{{...}}`、`${...}`、模板对象链或模板引擎报错。
- 已发现模板表达式可控，或存在明显模板上下文泄露迹象。

## 验证流程

1. 先做确定性表达式验证，例如 `{{7*7}}` 或等价语法。
2. 若表达式被执行，再尝试最小化对象链或上下文读取。
3. 若继续进入命令执行，必须记录“由模板注入触发”的完整链路。
4. 至少做一次负样本复检，确认不是页面静态回显或缓存结果。

## 误报防护

- 仅页面回显 payload、状态码变化或普通报错，不能确认 SSTI。
- 若证据更符合 Struts2/OGNL，不要误归因到 SSTI。
- 若模板注入进一步触发命令执行，根因类型仍优先标记为 SSTI。

## 报告要求

- 记录具体入口、模板表达式、响应中的确定性结果。
- 区分“类型确认”和“是否进一步拿到命令执行证据”。
